## 内容格式建议

```markdown
---
id: 'unique-id'
title: '文章标题'
excerpt: '文章摘要（150字以内）'
category: '分类名称'
readTime: 'X 分钟'
author: '作者姓名'
date: 'YYYY年MM月DD日'
image: 'https://example.com/cover.jpg'
tags: ['标签1', '标签2', '标签3']
tips:
  title: '小贴士标题（可选）'
  content: '小贴士内容（可选）'
  image: 'https://example.com/tips.jpg（可选）'
---

文章正文内容...
```

```markdown

```

## 实施步骤建议

1. **内容审计**：梳理知识库中所有可迁移内容（已确认 15 期双周刊存在）
2. **建立内容清单**：创建待迁移内容的 Excel/表格清单
3. **分批迁移**：按优先级分阶段迁移内容，先从双周刊开始
4. **审核校对**：每批内容迁移后进行审核
5. **链接检查**：确保所有内部链接和图片正常工作
6. **SEO 优化**：为重要页面添加元数据和描述

## 注意事项

1. **版权问题**：确保所有迁移内容都有正确的版权和授权
2. **图片处理**：将外部图片下载到本地或使用 CDN
3. **链接更新**：更新所有内部链接指向新网站
4. **格式统一**：确保所有内容使用统一的 Markdown 格式
5. **测试验证**：每个页面迁移后进行功能和显示测试
